Department of Education

UX/UI Design
A project to update the agency's website

General Information

ABOUT THE PROJECT

The goal of this project was to analyze the current website for the U.S. Department of Education. The focus of the project was to evaluate general usability by researching the UI pain points that users experience while interacting with the website and creating a responsive web redesign for the agency.

THE PROBLEM

The US Department of Education Website contains an overwhelming amount of information which is difficult to consume using the current website navigation which is not intuitive making it difficult for the users to find what they need.

THE SOLUTION

Re-design the US Department of Education website using a minimalistic and straight forward approach organizing the information in a streamlined manner guiding the user through a clear path to obtain the information they need.

Methodology

Where to start...

The journey into this project by gathering all the project requirements and organizing the activities that needed to be accomplished. We followed a strict methodology that started with planning our user research and finalizing with several iterations of user testing.

User Research

The plan...

A decision was made to start the user research by conducting in-person user interviews targeting some of the most common uses of the current website and analyzing the user pain points while accomplishing those activities. For this activity a usability testing document was created targeting three main objectives:

  • Find out if the users were able to successfully locate the requirements to apply for financial assistance
  • Evaluate how easy it was for the user to find the eligibility requirements
  • Determine if the users were able to start the financial aid application process

From the data collected, some trends emerged which were included in the following prioritization matrix:

User insights...

The data collected gave us insights into the major problems that users were facing:

  • All the information contained on the website which did not seem to be organized in an easy to understand manner
  • Small font sizes and confusion on overall information organization
  • Clear guidelines on how to navigate the website go get to the most common tasks

Definition and Planning

A clearer path...

After performing ideation and brainstorming exercises such as cart sorting on the current website, the main categories of information emerged which were represented in a site map showing a more cohesive information architecture, which showcased:

  • A primary level consisting of five main categories of information which would become the top navigation:
    Financial Aid, Repayment of Aid, Teachers & K-12 Students, Laws & Regulations and Data & Research
  • A secondary level containing organized information related to each parent main category
  • A footer category containing information related to the agency itself, ways to connect, and help for the users

What was achieved...

Organizing the data differently brought the following advantages:

  • Data duplication was going to be eliminated almost completely
  • Content was organized prioritizing what users needed most from the site
  • Information hierarchy was clearer

Prototyping

The solution starts to emerge...

Based on the users pain points discovered during the research phase, and following a newly design site map, the next step was to create low fidelity prototypes.

Given the time constraints and since the information to work from was very clear, instead of creating low fidelity prototypes and have users test them, a decision was made to start with the medium - high fidelity prototyping and give that activity all the time possible. This way, the users could test on a prototype that was closer to reality.

High fidelity prototype

For the high fidelity prototype, an improved style guide was created. Some of the existing elements were maintained in order to ensure continuity, but new elements were added to give the site new life.

User Testing and Outcomes

Let's get some feedback...

In order to test if the goal of the project was met by organizing the information in a streamlined manner guiding the user through a clear path to obtain the information they need, we conducted in-person usability testing. By setting up two objectives, the idea was to see how easy it was now to navigate the website and get the tasks done.

The two objectives were:

  • Find out if the user is eligible for financial aid
  • Locate the physical address for the Department of Education
User Testing Key Findings

The users were very happy to see the improvements. The following were the key findings we were able to uncover:

  • The success rate for the testers was excellent. All reached the testing goals with 100% success rate.
  • I received very useful feedback on how to improve the prototypes, which I analyzed and prioritized for implementation. Feedback mostly focused design upgrades and not on navigation. A few items that I was able to implement were:
    • Increase font size even more on some areas for better visibility
    • Change the color of the top menu on the mobile version to make it similar to the desktop version
    • Distribute the footer space to avoid too much blank space

Conclusion and Opportunities

A job well done...

This project was a great learning experience for me as I was required to perform intense research while interpreting and organizing data in different ways in order to meet with users expectations. As I finished by high fidelity prototypes and conducted usability user testing, I learned to prioritize different requests in order of importance since the window to implement those requests was very short.

There are endless opportunities to iterate and make our projects better. The difficult thing is to let them go when it is time to do so. Our desire in trying to reach the “perfect” project has to be controlled and let way to a sense of contentment that providing a very good solution brings. We need to understand that a project can always be improved.
